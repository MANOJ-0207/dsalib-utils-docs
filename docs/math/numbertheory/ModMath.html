ModMath

Type: Utility Class

Description

ModMath is a utility class for performing safe and efficient modular arithmetic operations under a given modulus. Supports operations like addition, subtraction, multiplication, division, modular exponentiation, and modular inverse.

Assumes that the modulus mod is a positive integer, and for operations like division and inverse, it assumes that mod is prime (based on Fermat’s Little Theorem).

Method Summary

Return Type Method Description Time Complexity
N/A ModMath(int mod) Constructs a modular arithmetic utility with a given modulus. O(1)
int modAdd(a, b) Modular addition: (a + b) % mod O(1)
int modSub(a, b) Modular subtraction: (a - b) % mod O(1)
int modMul(a, b) Modular multiplication: (a * b) % mod O(1)
int modDiv(a, b) Modular division: (a / b) % mod using modular inverse O(log mod)
int modPow(base, exp) Modular exponentiation: (base^exp) % mod O(log exp)
int modInverse(a) Modular inverse of a: a⁻¹ % mod O(log mod)
int getMod() Returns the current modulus O(1)

Detailed Method Descriptions

modAdd(int a, int b)

Performs modular addition ensuring result lies in [0, mod - 1]. 


ModMath mm = new ModMath(7);
int sum = mm.modAdd(5, 6); // 4

modSub(int a, int b)

Computes (a - b + mod) % mod to safely handle negative values.


int diff = mm.modSub(2, 5); // 4

modMul(int a, int b)

Uses long casting internally to prevent overflow.


int prod = mm.modMul(3, 4); // 5

modDiv(int a, int b)

Performs modular division using Fermat’s Little Theorem:
a / b ≡ a × b-1 mod m, where m is prime. 


int result = mm.modDiv(10, 4); // result = 5, if mod = 7

modPow(int base, int exp)

Binary exponentiation for efficient computation of powers under modulus. 


int power = mm.modPow(3, 4); // 4^3 = 81 % 7 = 4

modInverse(int a)

Returns x such that (a × x) % mod == 1. Assumes mod is prime.


int inv = mm.modInverse(3); // = 5 when mod = 7

getMod()

Returns the modulus used in this instance.


System.out.println(mm.getMod()); // 7

Mathematical Notes

  • All results are normalized to [0, mod - 1]
  • Division and inverse only work properly if mod is a prime
  • Binary exponentiation reduces power computation to logarithmic time
  • Used heavily in combinatorics and number theory (e.g., Fermat, CRT)

Quick Example


ModMath mm = new ModMath(17);
System.out.println(mm.modAdd(8, 11));       // 2
System.out.println(mm.modSub(3, 9));        // 11
System.out.println(mm.modMul(5, 4));        // 3
System.out.println(mm.modPow(3, 5));        // 5
System.out.println(mm.modInverse(7));       // 5
System.out.println(mm.modDiv(10, 4));       // 11
