NumberTheoryUtils

Type: Utility Class

Description

NumberTheoryUtils provides efficient static methods for common number theory operations such as GCD, LCM, the Extended Euclidean Algorithm, and Modular Inverse.

These functions are fundamental for modular arithmetic, cryptography, and algorithmic number theory.

Method Summary

Return Type Method Description Time Complexity
int gcd(int a, int b) Computes the Greatest Common Divisor of a and b O(log min(a, b))
int lcm(int a, int b) Computes the Least Common Multiple of a and b O(log min(a, b))
GCDTriplet extendedGCD(int a, int b) Finds x, y such that a·x + b·y = gcd(a, b) O(log min(a, b))
int modInverse(int a, int mod) Computes modular inverse of a under modulus mod O(log mod)

Detailed Method Descriptions

gcd(int a, int b)

Computes the greatest common divisor of two integers using the Euclidean algorithm. 


int result = NumberTheoryUtils.gcd(48, 18); // result = 6

lcm(int a, int b)

Computes the least common multiple using the formula:
lcm(a, b) = abs(a × b) / gcd(a, b).

Overflow-safe using long internally. Throws an exception if the result exceeds Integer.MAX_VALUE. 


int result = NumberTheoryUtils.lcm(12, 15); // result = 60

extendedGCD(int a, int b)

Computes coefficients x, y and gcd such that a·x + b·y = gcd(a, b).

Useful for finding modular inverses or solving Diophantine equations. 


NumberTheoryUtils.GCDTriplet result = NumberTheoryUtils.extendedGCD(30, 20);
// result.gcd = 10, result.x = 1, result.y = -1

modInverse(int a, int mod)

Computes the modular inverse of a under modulus mod using the extended Euclidean algorithm.

Valid only when gcd(a, mod) == 1. Throws an exception otherwise. 


int inverse = NumberTheoryUtils.modInverse(3, 11); // inverse = 4, since (3 × 4) % 11 == 1

Nested Class - GCDTriplet

Helper class to hold the result of the Extended Euclidean Algorithm.

  • int gcd: The greatest common divisor
  • int x: Coefficient of the first number
  • int y: Coefficient of the second number

NumberTheoryUtils.GCDTriplet triplet = NumberTheoryUtils.extendedGCD(10, 6);
System.out.println(triplet.gcd); // 2
System.out.println(triplet.x);   // -1
System.out.println(triplet.y);   // 2

Notes

  • All methods are static and side-effect free.
  • Recommended for use in modular inverse problems, CRT, and number theory applications.
  • Handles edge cases like zero and negative inputs.

Quick Example


System.out.println(NumberTheoryUtils.gcd(42, 30));         // 6
System.out.println(NumberTheoryUtils.lcm(12, 18));         // 36
System.out.println(NumberTheoryUtils.modInverse(3, 7));    // 5
