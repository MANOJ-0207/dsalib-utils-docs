PrimeUtils

Type: Utility Class

Description

PrimeUtils provides efficient static methods for prime number generation, factorization, and testing. This includes Sieve of Eratosthenes, Segmented Sieve, Smallest Prime Factor (SPF) computation, and trial division checks.

Method Summary

Return Type Method Description Time Complexity
boolean[] simpleSieve(int n) Computes primality for all numbers up to n using Sieve of Eratosthenes O(n log log n)
int[] sieveForPrimeFactorization(int n) Computes smallest prime factor (SPF) for each number ≤ n O(n log log n)
List<Integer> getPrimeFactors(int x, int[] spf) Returns list of prime factors of x using SPF O(log x)
boolean[] segmentedSieve(int m, int n) Finds all primes in the range [m, n] using Segmented Sieve O((n - m + 1) log log n)
boolean isPrime(int x) Checks whether x is prime by searching for a factor before √x O(√x)

Detailed Method Descriptions

simpleSieve(int n)

Returns a boolean array of size n + 1 where each index indicates whether the number is prime. 


boolean[] isPrime = PrimeUtils.simpleSieve(10);
// true values at indices 2, 3, 5, 7

sieveForPrimeFactorization(int n)

Precomputes smallest prime factor (SPF) for all numbers ≤ n. 


int[] spf = PrimeUtils.sieveForPrimeFactorization(20);
// spf[15] = 3, since 3 is the smallest prime factor of 15

getPrimeFactors(int x, int[] spf)

Decomposes a number into its prime factors using a precomputed SPF array. 


int[] spf = PrimeUtils.sieveForPrimeFactorization(100);
List<Integer> factors = PrimeUtils.getPrimeFactors(60, spf);
// Output: [2, 2, 3, 5]

segmentedSieve(int m, int n)

Finds all prime numbers in the range [m, n] using the Segmented Sieve technique. Useful for large ranges where regular sieve would be too memory-intensive. 


boolean[] rangePrime = PrimeUtils.segmentedSieve(10, 20);
// rangePrime[0] = false (10), rangePrime[1] = true (11), ...

isPrime(int x)

Checks whether a given number is prime using simple trial division up to √x. 


boolean result = PrimeUtils.isPrime(29); // true

Notes

  • All methods are static and side-effect free.
  • getPrimeFactors requires a precomputed SPF array (use sieveForPrimeFactorization).
  • segmentedSieve is best suited for large prime ranges.
  • Handles invalid inputs with clear exceptions.

Quick Example


boolean[] isPrime = PrimeUtils.simpleSieve(30);
System.out.println(isPrime[29]); // true

int[] spf = PrimeUtils.sieveForPrimeFactorization(100);
System.out.println(PrimeUtils.getPrimeFactors(60, spf)); // [2, 2, 3, 5]

boolean[] segment = PrimeUtils.segmentedSieve(50, 60);
// segment[1] = true => 51 is prime? false (but check for others)

System.out.println(PrimeUtils.isPrime(97)); // true
