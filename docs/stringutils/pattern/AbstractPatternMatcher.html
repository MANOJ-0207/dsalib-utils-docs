AbstractPatternMatcher

Type: Abstract Class

Description

AbstractPatternMatcher is an abstract base class that implements the PatternMatcher interface. It provides shared validation and input normalization logic for pattern matching algorithms using the template method pattern. Subclasses only need to implement the core matching logic in the doSearch(char[], char[]) method.

Implemented Methods

Return Type Method Description Implementation Details
List<Integer> search(String text, String pattern) Validates string inputs and converts to char arrays Null validation, converts strings to char[], delegates to doSearch()
List<Integer> search(char[] text, char[] pattern) Validates char array inputs Null validation for arrays, delegates to doSearch()

Abstract Methods

Return Type Method Description
List<Integer> doSearch(char[] text, char[] pattern) Core pattern matching logic to be implemented by subclasses

Available Concrete Implementations

  • KMPMatcher - Knuth-Morris-Pratt algorithm implementation
  • RabinKarpMatcher - Rabin-Karp rolling hash algorithm
  • ZAlgorithmMatcher - Z-algorithm-based linear time pattern matching

Basic Usage


// Example of extending AbstractPatternMatcher
public class MyPatternMatcher extends AbstractPatternMatcher {
    @Override
    protected List<Integer> doSearch(char[] text, char[] pattern) {
        // Implement your pattern matching algorithm here
        // Input validation is already handled by the base class
        List<Integer> matches = new ArrayList<>();
        // ... matching logic ...
        return matches;
    }
}

// Usage
PatternMatcher matcher = new MyPatternMatcher();
List<Integer> results = matcher.search("hello world", "world");
